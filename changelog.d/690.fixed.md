- **Idle codex panes with no ghost-text no longer misclassify as `StateUnknown`** (#690). An empty codex composer renders `› ` (prompt sentinel + trailing space); `tmux capture-pane -p` strips the trailing space, leaving a bare `›`, which the sentinel match (`"› "`, space included) missed — so a genuinely-idle pane fell through to `StateUnknown` ("prompt sentinel not found in any row"). That is paste-unsafe, so the mailman deferred *all* inbound delivery to the chamber until a ghost-text placeholder returned. The sentinel match now tolerates the strip (matches the right-trimmed sentinel when the cursor sits at the full sentinel column), across both the cursor-aware and cursor-less classification paths. Claude is unaffected — its sentinel ends in NBSP (U+00A0), which capture-pane does not strip.
