- **Codex delivery verify is now load-adaptive, so heavy pastes submit within their own mailman cycle** (#674). Building on #610 (which drains a stuck collapsed paste across cycles), the post-Enter verify loop now reads the frame-delta between its existing per-poll captures to adapt to codex load: (1) *stability-gated resubmit* — the resubmit Enter is held while the pane is still redrawing (a mid-render Enter is eaten) and fired only once the frame settles with the collapse marker still present, so it lands submit-ready; (2) *load-scaled budget* — past the base retry schedule the loop keeps polling while the delivery is objectively still mid-ingest (collapse marker present and frame changing), up to a bounded ceiling, so a heavy paste submits in-cycle instead of exhausting the fixed budget and deferring to the next visit. A final best-effort Enter preserves the pre-existing guarantee that a stuck paste always gets at least one resubmit. Claude is unaffected — it has no collapse marker, so no resubmit is gated and the extension never triggers; the base schedule runs exactly as before. The load-reliability property is validated by post-deploy observation under real codex load (sustained high-throughput windows), not point-in-time tests.
