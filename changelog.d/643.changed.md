Session-id discovery now reads a single adapter-neutral env var, `TMUX_TELL_SESSION_ID`, for every adapter — Claude's former native `CLAUDE_CODE_SESSION_ID` self-discovery is dropped (#643). The native path latched the *launch-era* session id from long-lived MCP-server children (a Claude pane's process tree carries multiple `CLAUDE_CODE_SESSION_ID` values) and was fragile to MCP-server lifecycle; the launch wrappers now mint one UUID and inject it tree-wide as `TMUX_TELL_SESSION_ID` (passed at `register --session-id` and `--setenv`'d into the launched process environ), the same mechanism codex + aichat already use (#626 Phase 2). One uniform var across all three adapter sources. A pane with no injected var (a raw non-wrapper `claude --resume` launch) self-discovers nothing and resolves via the name-based fallback, as before. Requires the launch wrapper to inject the var (alcatraz-infra#105); existing Claude chambers acquire it on their next wrapper launch.
