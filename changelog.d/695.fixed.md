- **The load-adaptive verify loop no longer panics on an empty retry schedule** (#695). #674's load-adaptive extension (`ceiling = len(retryDelays) + maxLoadAdaptiveExtraAttempts`) made the patient-tail backoff index reachable even with no schedule, so `SetRetrySchedule([])` would drive `retryDelays[-1]` and panic. Unreachable in production (the schedule is only ever set from `DeriveRetrySchedule`, always non-empty), but a self-introduced regression from #674 — the pre-#674 loop never indexed an empty schedule. Guarded: an empty schedule now stops after the immediate verify poll (the pre-#674 empty-schedule behavior); non-empty schedules are unchanged.
