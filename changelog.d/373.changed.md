`doctor` now parses its flags with the standard `flag` package like every other subcommand (#373), so `--format text` (space form), `--format=json`, `-h`/`--help`, and consistent rejection of unknown flags/args all work the same way they do elsewhere — previously it hand-rolled a `--format`-only loop. Separately, the `/proc`-derived DB-binding report (shown by `doctor` and `whoami db`) no longer conflates "couldn't read `/proc/<pid>/fd`" (a permission denial or an exited-mid-walk race) with "no `.db` handle open": an unreadable fd directory now surfaces as `could not read /proc/<pid>/fd: <err>` so the operator sees the binding is *unknown* rather than assuming the process has no DB open.
