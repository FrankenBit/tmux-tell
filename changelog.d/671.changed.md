`uninstall.sh` now mirrors `install.sh`'s mode split (#671, the companion to #636): a bare `./uninstall.sh` is a **user-space uninstall** that needs no root (removes the binary + template from `~/.local`), and `sudo ./uninstall.sh --system` undoes a root install under `/usr/local`. It also fixes a pre-existing rename gap — the script targeted the legacy `claude-msg` binary and `claude-mailman@` template, leaving the canonical `tmux-tell-<adapter>` binary and `tmux-tell-<adapter>-mailman@` template behind; it now removes the canonical artifacts plus their `tmux-msg-<adapter>` / `claude-msg` deprecation aliases (a `-L` check catches a dangling alias symlink too). A new `--adapter=NAME` flag (claude default, or codex) selects which adapter to remove, and the `--system`/user-space split keys the PREFIX default, the EUID guard (user-mode rejects root; `--system` requires it), and the `systemctl --user` privilege-drop — exactly as `install.sh` does. Reverting a user-space install by hand (`rm ~/.local/bin/tmux-tell-*` + the template) remains documented in the README as the no-script path.
