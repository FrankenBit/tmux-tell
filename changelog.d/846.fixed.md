`deliver_after=resume` rows now also auto-fire on a **self-typed** `/compact`, not only a bus-delivered one. #843 covered a reset that arrived as a control row (`isSessionResetControl`); a chamber that types `/compact` directly at its pane leaves no such row, so its staged resume rows still rotted — the honest residual #843 disclosed (4 of 7 stuck rows). The mailman now edge-detects the same `last_self_compact_at` compaction signal that #285 PR2 counts, against a **second** mailman-owned watermark (`resume_promoted_at`, deliberately distinct from `self_compact_counted_at` so the promote and the respawn-shrink counter never consume each other's edge), and promotes the agent's resume rows at that edge — i.e. at the resume moment, when the freshly-compacted oriented context is arriving, which is exactly what `deliver_after=resume` stages for. **Dormant until deployed:** the signal it reads is only populated once the adapter's `PostCompact` → `note-compact` hook is wired fleet-wide (alcatraz-infra#391) — today it is stamped for one chamber only — so this ships inert (like #285 PR2) and activates when the hook install lands and a binary carrying it deploys. Promotion is unbounded and un-age-filtered by design, preserving #227's `flush_deferred` contract; because it reaches precisely the oldest rows, the one-time operator drain at deploy noted on #843 applies.
