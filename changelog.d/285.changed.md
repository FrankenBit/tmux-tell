Self-compact detection for bounded post-shrink respawn (#285): a chamber's own `/compact` (auto or manual) now feeds the respawn counter, not just a bus-delivered `/clear`. Wire your adapter's post-compaction hook — Claude Code's `PostCompact` or Codex's `[[hooks.PostCompact]]` (matcher `manual|auto`) — to run `tmux-tell-<adapter> note-compact`. That helper records a self-compact signal for the calling chamber; the mailman counts it on its self-observation pass and stays the sole writer of the counter (the hook only ever writes the signal, so it can't race the count). Opt-in per chamber, still disabled by default (respawn threshold 0). Adapter classes without hook infrastructure are tracked as follow-up #685.
