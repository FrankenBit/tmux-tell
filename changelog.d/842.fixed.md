Fixed Claude paste deliveries that landed in the composer but never submitted, leaving the operator to press Enter by hand. The `#401`/`#674` resubmit machinery was structurally unreachable on Claude: its gate keyed on `PasteCollapseMarker`, which `ClaudePaneProfile` leaves empty, so the "is our paste still sitting unsubmitted?" predicate was **always false** for Claude — `Deliver` sent exactly one Enter and the verify loop then only re-CAPTURED until the budget expired, returning `ErrUnverifiedDelivery` (a soft success). Nothing ever re-sent the Enter. A new `PasteEvidenceMarker` profile field splits the two questions that field had conflated: `PasteCollapseMarker` now means only "this adapter collapses pastes, so normalize the body for it" (it gates `normalizeCollapsePaste`), while `PasteEvidenceMarker` is the string that positively identifies one of OUR pastes in the live composer. Claude supplies `[Pasted text` (its composer collapses a large paste to `[Pasted text #N +M lines]` as of Claude Code ≥2.1); codex reuses its existing collapse marker, so codex behavior is unchanged. Keeping the fields separate is load-bearing: giving Claude a `PasteCollapseMarker` would have been the tempting one-liner and would have silently started mutating Claude paste bodies with the `#533` codex sign-off accommodation. The predicate deliberately keys on a marker rather than on "the input is non-empty" — a cursor-anchored not-cleared test also matches a half-typed operator draft, and resubmitting on that submits the operator's draft (the pre-existing Claude operator-draft test caught exactly that during development). Measurement against `claude` 2.1.218 also **refuted** the originally-hypothesised cause: frame-settle after a paste is flat at ~38–40 ms across a 40× byte range (181 B → 7279 B) because the collapse makes render cost roughly constant, and a 3457 B / 38-line paste submitted 3/3 at the production 500 ms settle delay — so the failure is not paste size and no size-scaled settle was added. The true trigger for the observed incident (a post-`/compact` staged resume) remains uncharacterised; this change is cause-agnostic recovery, draining an objectively-unsubmitted paste whatever prevented the first Enter.
