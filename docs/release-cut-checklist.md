# Release-cut docs-coherence checklist

Operator-facing truth about tmux-tell lives across several surfaces with
**different review nets**. A code change is reviewed in its own PR; the docs that
describe it may live somewhere that PR never touches. This checklist makes those
off-PR-net surfaces **visible at cut time** so a change doesn't ship coherent in
code but stale in the docs an operator actually reads. (Worked anchor: the
`tmux-msg` → `tmux-tell` rename landed in code while doc-prose, BookStack pages, and
chamber instructions drifted behind it — #495.)

**What this is — and isn't.** This is a **salience** mechanism, not a machine gate.
It surfaces the surfaces at cut time so the cut-driver verifies them by hand; it does
**not** mechanically enforce coherence (no CI fails if BookStack drifts). Naming
that honestly matters: the checklist's value is that it makes the off-net surfaces
*available for an opportunistic check*, not that it guarantees them. If vigilance
proves insufficient, a CI/audit tool is the deferred follow-up (Layer 3). The
release-prep PR body carries a compact checkbox version (emitted by `release.yml`);
**this doc is the substrate-of-record for how to verify each item.**

## The surfaces, by review-net

| Surface | Review-net | Verify | If it drifted |
|---|---|---|---|
| **In-repo docs** — `README.md`, `docs/*.md` (except ADRs), `CONTRIBUTING.md`, `cmd/*/README.md`, `CHANGELOG.md` | **PR-net** — in the diff, reviewer-visible | `git grep '<changed-identifier>' -- '*.md' ':!docs/adr/*'` | fix in this PR |
| **ADRs** — `docs/adr/*` | **frozen** | n/a | leave — point-in-time records, never retro-edited (see `docs/adr/README.md`) |
| **BookStack** — Service Inventory (page 88), Release & Deploy Procedure (page 193), the *tmux-tell* book | **API** — *not* in any PR diff | curl/MCP the pages for the changed identifier | update via the BookStack API in the cut work-cycle (Herald-token; see `/srv/CLAUDE.md` § Documentation) |
| **alcatraz-infra repo** — `/srv/CLAUDE.md` (service inventory, sudoers comments, observability refs) | **separate repo** | `grep '<changed-identifier>' /srv/CLAUDE.md` | a separate alcatraz-infra commit/PR — **not** this repo's cut |
| **Sister chamber `CLAUDE.md`** — `/srv/claude/<chamber>/CLAUDE.md` | **cross-authority** | `grep -l '<changed-identifier>' /srv/claude/*/CLAUDE.md` | **flag/ping the affected chamber — do not edit** (each chamber owns its own) |

Only the first tier is caught by ordinary PR review. The other four are off the
PR/CI net — which is exactly why they drift, and exactly why they need a cut-time
prompt.

## Per-surface notes

- **In-repo docs** — the cheap, reviewer-visible tier. The grep is your friend: if
  the cut renamed/retired an identifier, flag, path, env var, or command, sweep
  `docs/` + the READMEs for the old form. Distinguish *live* references (flip) from
  *deprecation-doc / historical / migration-recipe* references (keep) — see the
  rename PR #483 for the flip-vs-preserve discipline.
- **ADRs** — frozen by policy. An ADR records a decision *at its time*; it keeps the
  then-current name. Do not retro-edit; `docs/adr/README.md` carries the historicity
  note.
- **BookStack** — the easiest surface to forget because it's API-only (never in a
  git diff, never in CI). Pages **88** (Service Inventory) and **193** (Release &
  Deploy Procedure) carry repo URLs, binary names, and service references that go
  stale on a rename/path/port change. Update with the Herald BookStack token (the
  curl-API, not the shared-MCP, for own attribution).
- **alcatraz-infra `/srv/CLAUDE.md`** — a *separate repo*. Service-inventory rows,
  sudoers-rule comments, and observability references can name tmux-tell surfaces.
  Fix in an alcatraz-infra commit/PR; do **not** fold it into the tmux-tell cut.
- **Sister chamber `CLAUDE.md`** — cross-chamber authority. Each chamber owns its
  own instructions; a cut-driver who finds a stale reference **flags/pings that
  chamber**, it does not edit another chamber's file (Phase-4 scope-fence).

## How this composes

- Sibling to the **#427 prelude check** (CONTRIBUTING § Release cuts, step 2): that
  guards the *release body*; this guards the *surrounding docs*.
- The **release-prep PR body** (generated by `release.yml`) carries the compact
  checkbox version; this doc is the detail (rationale / verify / drift-action) —
  the same two-layer shape as the CHANGELOG convention (#454).
- **Every** release-prep PR carries it; items are **N/A-able** — a cut with no
  operator-facing surface change ticks them off in seconds. Don't skip it for
  "small" cuts: *"small-cut" exemptions are exactly how drift re-enters* (which is
  why the gate is unconditional rather than change-triggered).
