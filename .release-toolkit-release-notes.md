### Added

- Bounded post-shrink chamber respawn (#285, PR1): the mailman can now proactively restart a chamber's process after N counted context-shrink events, releasing the heap `/compact` and `/clear` can't. This complements the memory-cap wrapper's host-protection (alcatraz-infra#50) with a *graceful* lever — a clean `/exit` + `respawn-pane -k` (original cmdline, so the memory-cap wrapper is preserved) on the chamber's own idle cadence, instead of an abrupt OOM-kill and a manual relaunch. Opt-in and disabled by default: set the per-chamber threshold with `set-respawn-after-shrinks --name <chamber> <N>` (0 = off; the running mailman picks up the change next cycle, no restart). This first cut counts a bus-delivered `/clear` as the shrink trigger; the respawn re-establishes session identity (#626) and threads through the #227 deferred-delivery flow so nothing lands on the dying process. Self-compact detection (PostCompact hook + transcript polling) is a tracked follow-up.
- `TMUX_TELL_DEBUG` env var (#681): set it to any non-empty value to turn on verbose debug logging (to stderr) on the substrate's diagnostics-heavy paths; empty or unset keeps them quiet. The first consumer is `ping`'s evidence-gather (#365 nit2) — behind an UNREACHABLE ping, each best-effort probe that drops an error (agent lookup, queue depth, state resolve) now logs a line under the gate instead of degrading silently to a zero value. Scope is deliberately narrow (only genuinely-diagnostic paths gate on it, not every dropped error); the gate is the `internal/debug.Enabled()` helper and is documented in `docs/reference.md` (§ Debugging) and `docs/agent-manual.md`.

### Changed

- Self-compact detection for bounded post-shrink respawn (#285): a chamber's own `/compact` (auto or manual) now feeds the respawn counter, not just a bus-delivered `/clear`. Wire your adapter's post-compaction hook — Claude Code's `PostCompact` or Codex's `[[hooks.PostCompact]]` (matcher `manual|auto`) — to run `tmux-tell-<adapter> note-compact`. That helper records a self-compact signal for the calling chamber; the mailman counts it on its self-observation pass and stays the sole writer of the counter (the hook only ever writes the signal, so it can't race the count). Opt-in per chamber, still disabled by default (respawn threshold 0). Adapter classes without hook infrastructure are tracked as follow-up #685.
- `doctor` now parses its flags with the standard `flag` package like every other subcommand (#373), replacing a hand-rolled `--format` loop — a consistency fix (its `--format` handling, space/`=` forms, and unknown-flag/argument rejection already worked). The one observable change: `doctor --format` with a *missing* value now errors instead of silently defaulting to `text`, and `-h`/`--help` prints the standard flag usage. Separately, the `/proc`-derived DB-binding report (shown by `doctor` and `whoami db`) no longer conflates "couldn't read `/proc/<pid>/fd`" (a permission denial or an exited-mid-walk race) with "no `.db` handle open": an unreadable fd directory now surfaces as `could not read /proc/<pid>/fd: <err>` so the operator sees the binding is *unknown* rather than assuming the process has no DB open.
- `install.sh` now defaults to a **user-space install** that needs no root (#636): the binary goes to `~/.local/bin/tmux-tell-<adapter>` owned by you, the systemd template to `~/.config/systemd/user/`, and nothing is root-owned — so you can clone the repo and `./install.sh` to evaluate the tool without granting `sudo` to an unaudited binary. The historical root install (binary root-owned under `/usr/local/bin`, on the system `PATH`) is now opt-in via `./install.sh --system`; that mode is byte-for-byte the old behavior. As part of the user-space default the installer now rewrites the systemd unit's `ExecStart` to the actual install prefix (the shipped template hardcodes `/usr/local/bin`, so a non-default prefix previously produced a unit pointing at a missing binary), and `loginctl enable-linger` is best-effort in user mode (the bus works for the current session; re-run with `sudo` for boot-persistence). Reverting a user-space install is a plain `rm` of the user binary + template (documented in the README). The alcatraz deploy chain (`make install`, the deploy workflow) pins `--system` to preserve the existing system-wide behavior. `uninstall.sh` still targets `--system` installs; mirroring the user-space default is tracked as #671.
- `uninstall.sh` now mirrors `install.sh`'s mode split (#671, the companion to #636): a bare `./uninstall.sh` is a **user-space uninstall** that needs no root (removes the binary + template from `~/.local`), and `sudo ./uninstall.sh --system` undoes a root install under `/usr/local`. It also fixes a pre-existing rename gap — the script targeted the legacy `claude-msg` binary and `claude-mailman@` template, leaving the canonical `tmux-tell-<adapter>` binary and `tmux-tell-<adapter>-mailman@` template behind; it now removes the canonical artifacts plus their `tmux-msg-<adapter>` / `claude-msg` deprecation aliases (a `-L` check catches a dangling alias symlink too). A new `--adapter=NAME` flag (claude default, or codex) selects which adapter to remove, and the `--system`/user-space split keys the PREFIX default, the EUID guard (user-mode rejects root; `--system` requires it), and the `systemctl --user` privilege-drop — exactly as `install.sh` does. Reverting a user-space install by hand (`rm ~/.local/bin/tmux-tell-*` + the template) remains documented in the README as the no-script path.

### Fixed

- Bus messages to codex chambers no longer stack as unsubmitted text in the composer when codex is under load (#610). The mailman delivers via `paste-and-enter`; under high context/output load codex is still ingesting and collapsing a large paste (`[Pasted Content …]`) when the submit-Enter fires, so the per-delivery verify budget exhausts with the paste still unsubmitted. The **next** delivery then pasted on top of it — the #616 pre-paste race check is cursor-anchored and can't see a stuck collapsed paste (codex parks the cursor on an empty sub-line so the input row reads "cleared"), so it was scoped to operator drafts and left the collapse marker to the #401 resubmit machinery, which under load runs out of budget. Messages accumulated into one corrupted ~15-line composer that the operator had to submit by hand. The pre-paste check now also detects a stuck collapsed paste (the adapter's `PasteCollapseMarker` in the bottom-most input block): rather than paste onto it, the mailman fires one resubmit Enter to drain it — extending the #401 resubmit across mailman cycles, not just within one delivery's budget — and reverts the current message to queued for a later cycle. Stacking/corruption becomes clean serialization, and the stuck paste drains without the operator as the backstop. Codex-specific by config (Claude has no collapse marker, so it is unaffected). Reduces the upstream cause of #609 (a stuck composer keeps the prompt row non-empty, so the classifier reads `unknown` and the mailman defers all delivery — fewer stuck composers, fewer dark chambers). The deeper load-adaptive submit-reliability work (stability-gated resubmit, load-scaled budget) is tracked separately as a follow-up.
